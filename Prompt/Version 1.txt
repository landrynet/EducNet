# PROMPT – DÉVELOPPEMENT DU SYSTÈME DE GESTION SCOLAIRE MULTI-SITE (VERSION 1)

Tu es un Architecte Logiciel Senior, Expert Django Full Stack, UI/UX Designer et Expert en cybersécurité.

Ta mission est de concevoir un système de gestion scolaire professionnel, moderne, sécurisé, modulaire et évolutif.

Ne produis pas un prototype.

Produis une application prête à évoluer vers une solution professionnelle.

# Architecture

Le projet doit être organisé en applications Django indépendantes.

Chaque module doit être séparé.

Le système doit être facilement maintenable.

Ne jamais utiliser le Django Admin pour gérer les données métier.

Créer un tableau de bord entièrement personnalisé.

---

# Type d'application

Application Web Multi-Site (Multi-Tenant)

Chaque établissement possède son propre espace.

Une école ne peut jamais accéder aux données d'une autre école.

Toutes les données métier doivent être liées à une école.

# Fonctionnement

Le Super Admin crée une école.

Le système crée automatiquement un espace dédié.

Le Super Admin crée le premier compte Admin École.

L'Admin École se connecte.

Lors de la première connexion :

* changement obligatoire du mot de passe ;
* finalisation du profil ;
* accès au tableau de bord.

Ensuite, l'Admin École peut créer :

* les secrétaires ;
* les enseignants ;
* les comptables.

Chaque utilisateur doit uniquement voir les modules qui lui sont attribués.

Le système doit appliquer un contrôle d'accès basé sur les rôles et les permissions.
