# 2. INFORMATIONS SAISIES PAR LE SUPER ADMIN

Lors de la création de l'école, le Super Admin saisit uniquement les informations initiales nécessaires :

* Nom de l'école.

* Ville.
* Pays.
* Téléphone de contact initial.
* E-mail de contact initial.

Le Super Admin ne doit pas avoir à remplir toutes les informations détaillées du futur administrateur de l'école.

# 3. CRÉATION AUTOMATIQUE DE L'ACCÈS INITIAL

Après la création de l'école, le système doit générer automatiquement un compte initial pour le responsable ou l'administrateur de l'école.

Le compte initial doit comporter :

* Un identifiant unique.
* Un mot de passe temporaire.

Exemple :

```text
Identifiant :
admin@ecole.com

Mot de passe temporaire :
Xxxxxxxx
```

L'identifiant peut être un e-mail ou un autre identifiant unique défini par le système.

Le Super Admin doit pouvoir transmettre ces informations au responsable de l'école de manière appropriée.

⚠️ Le mot de passe temporaire ne doit pas être affiché inutilement dans les listes ou les pages de détails de l'école.

# 4. PREMIÈRE CONNEXION

Lors de la première connexion, l'administrateur de l'école doit obligatoirement saisir :

```text
Identifiant
+
Mot de passe temporaire
```

Exemple :

```text
E-mail / Identifiant
[ admin@ecole.com ]

Mot de passe temporaire
[ ************** ]

[ SE CONNECTER ]
```

Le système doit vérifier :

1. Que l'identifiant existe.
2. Que le compte correspond à l'école concernée.
3. Que le mot de passe temporaire est correct.
4. Que le compte est bien en état de première connexion.

Après authentification réussie, le système doit détecter qu'il s'agit de la première connexion.

# 5. CHANGEMENT OBLIGATOIRE DU MOT DE PASSE

Après la première connexion avec :

```text
Identifiant
+
Mot de passe temporaire
```

l'administrateur doit obligatoirement être redirigé vers une page de changement de mot de passe.

Il ne doit pas pouvoir accéder au dashboard normal avant d'avoir terminé cette étape.

La page doit contenir :

```text
Identifiant :
[ admin@ecole.com ]

Mot de passe temporaire :
[ ************** ]

Nouveau mot de passe :
[                 ]

Confirmer le nouveau mot de passe :
[                 ]

[ CHANGER LE MOT DE PASSE ]
```

Le système doit vérifier :

* Que le nouveau mot de passe respecte les règles de sécurité.
* Que les deux nouveaux mots de passe correspondent.
* Que le mot de passe temporaire est valide.
* Que l'utilisateur est bien autorisé à effectuer cette modification.

Après validation :

```text
Mot de passe temporaire
        ↓
Invalidé
        ↓
Nouveau mot de passe
        ↓
Activé
```

Le mot de passe temporaire ne doit plus être utilisable après le changement.
