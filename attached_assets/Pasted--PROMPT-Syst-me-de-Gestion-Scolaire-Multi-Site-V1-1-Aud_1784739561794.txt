* Plusieurs écoles.
* Des utilisateurs.
* Les différents rôles existants.
* Des comptes administrateurs.
* Des enseignants.
* Des secrétaires.
* Des comptables.
* Les données nécessaires au fonctionnement et au test de la V1.

Le script doit être :

* Reproductible.
* Facile à exécuter.
* Documenté.
* Sécurisé.
* Utilisable uniquement dans l'environnement de développement ou de test.

Créer un fichier ou une structure adaptée au framework utilisé.

`seed_test_data.py`

Le script doit permettre de recréer facilement un environnement de test propre.

# 5. TEST COMPLET DE LA V1

Après la création des données de test, tester le système avec plusieurs comptes et plusieurs écoles.

## Authentification

Vérifier :

* Connexion.
* Déconnexion.
* Mauvais mot de passe.
* Session expirée.
* Accès non autorisé.
* Protection des pages privées.

## Rôles et permissions

Vérifier que chaque rôle :

* Voit uniquement les fonctionnalités autorisées.
* Ne peut pas accéder aux pages interdites.
* Ne peut pas modifier des données non autorisées.

## Multi-Site

Vérifier impérativement qu'un utilisateur d'une école ne peut jamais :

* Voir les données d'une autre école.
* Modifier les données d'une autre école.
* Supprimer les données d'une autre école.
* Accéder aux URLs privées d'une autre école.

L'isolation entre les établissements doit être stricte.

## Navigation

Vérifier :

* Toutes les routes.
* Tous les liens.
* Tous les boutons.
* Tous les formulaires.
* Les redirections.
* Les menus selon les permissions.

## Base de données

Vérifier :

* Les migrations.
* Les relations.
* Les contraintes.
* Les données de test.
* La cohérence des données.
